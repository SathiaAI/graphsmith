## Headline
v0.2.0 turns GraphSmith from a skill into **a skill that publishes a protocol.** It extracts GraphSmith's integrity and evolution behaviour into a versioned, schema-backed, conformance-testable specification — the **GraphSmith Protocol v0.1-draft** — and ships the machinery that implements it: self-healing, gated learning (staged-only), an assurance/red-team harness, and evidence-carrying verification with a CI action and badge.

## What's new
- **GraphSmith Protocol v0.1-draft** (`GRAPHSMITH-PROTOCOL.md`, `SPEC-CHANGES.md`): five invariants (I1–I5), the capability profiles **R/E/B/T/G/Q/X**, and the four-gate promotion pipeline, with published JSON schemas and named attacker classes (A1–A6; A6 out-of-scope, stated).
- **Evidence-carrying verification:** `graphsmith verify --integrity | --profiles | --trust-model | --platform-probe | --selftest`. Profiles report `verified | unavailable | failed | not-applicable` — `unavailable` is never green; independent axes are never blended. Integrity exits `0/1/3` with failure domains (untrusted-input quarantine / evolvable-surface freeze / trusted-core halt).
- **Self-healing (staged-only)** and **gated learning (staged-only)** — code repairs and learned rules are staged for a human; the four gates and the typed event compiler keep untrusted content out of control flow.
- **Assurance harness (§17):** `test` / `lint` / `redteam` / `assure` + an external-tool seam; standing attack batteries (constitutional / TOCTOU / module-escape) gating in CI.
- **CI & supply chain:** a GitHub Action (`action.yml`) + GitLab template, a 3-OS verify matrix (Ubuntu/Windows/macOS × Node 18/22), PR-separation guard, docs-hygiene + honest-language lint, dogfooding, and the supply-chain model in `docs/SUPPLY-CHAIN.md`.
- **Evidence-carrying badge** (`badge.js`) + **property matrix** (`matrix.js`) + **Loop-3 shadow harness** (`shadow.js`) with the evaluator-stability memo (`docs/EVALUATOR-STABILITY.md`).

## Honest scope (unchanged, binding)
GraphSmith is a reliability, discipline, and **evidence** layer — not a security scanner, not a certification. The chaos harness tests exactly two properties (crash recovery; single, replay-verified execution of *recorded* effects, with loud halts when an external send's outcome is uncertain). A passing battery is a floor. Human gates on money/identity/private-data are load-bearing by design.
