# Platform Property Matrix

## Scope

**Tier-0 (agentless) capability profiles** — per-platform verification of core GraphSmith capabilities (R/E/B/T/G/Q/X).

> **Local dev snapshot (Wave 0).** Generated from a single real `verify --profiles` run on a Windows development checkout. On a dev checkout, `T` (release-verified) is legitimately `unavailable`, and `E`/`Q`/`X` need adapter/assurance/adversarial runs — all shown honestly as `unavailable`, never a pass. The **release matrix** (3-OS Ubuntu/Windows/macOS against the installed artifact) is regenerated by CI at tag time and supersedes this snapshot.

Agent-family matrix rows are added separately by the maintainer through per-agent verification runs (plan §16).

## Legend

| Status | Meaning |
|--------|---------|
| verified | Capability is verified on this platform |
| unavailable | Capability cannot be established on this platform (not a pass — honest gap) |
| failed | Capability verification failed |
| not-applicable | Capability does not apply to this platform |
